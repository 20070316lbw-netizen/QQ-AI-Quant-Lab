# 个人偏好与知识库 (Persona & Preferences)

## 核心原则
- **全中文交互**：在此项目的所有对话、方案设计、文档说明（如 implementation_plan, walkthrough 等）中，**必须**默认使用中文。
- **技术术语保留**：在不影响理解的前提下，核心技术术语（如 LangChain, LLM, V6.0 Architecture）可保留英文原名。
- **代码注释**：在修改代码时，尽量使用中文进行逻辑说明。

## 架构偏好 (V6.0+)
- **去博弈化**：优先考虑参数化、数学驱动的因子聚合模型，而非冗余的智能体对话。
- **极致响应**：关注系统响应延迟，通过减少不必要的 LLM 调用来优化性能。
- **量化可追溯**：决策必须由数据和公式推导，严禁 LLM 的话术修饰干扰最终结论。

## Git 与同步规则 (V6.1+)
- **GitHub 隔离**：生成的各种结果文件、`.agents/` 目录（尤其是 `persona.md` 和 `sync-persona.md`）默认**不向 GitHub 同步**，以保持公共仓库的纯度。
- **本地存档**：这些文件可以提交到本地 Git 进行历史存档，但在推送到 remote (GitHub) 时应予以隔离或排除。
- **DEMO 例外**：仅当为了生成 `.example` 或示范类文件进入仓库进行展示时，才允许同步相关产物。
